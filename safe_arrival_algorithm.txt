Safe Arrival Algorithmus (PlatzPilot)
===================================

Ziel
----
Fuer jeden Lernort wird aus den Wochenhistorien eine konservative Empfehlung
berechnet:

Empfohlene Ankunft bis HH:mm

Zusätzlich liefern wir:
- Peak ⌀ (frueh gewichteter Peak)
- Sicherheitsniveau
- Peak-Trend (wird frueher / spaeter / bleibt gleich)

Datenbasis
----------
- Historie: bis zu 2304 Punkte ab Mitternacht vor einer Woche bis jetzt
- Zeitauflösung: 5-Minuten-Bins
- Historische Modellierung nutzt seat-estimates + manualcount
  (manualcount ueberschreibt seat-estimate bei gleichem Zeitstempel)
- Aktuelle Anzeige priorisiert manualcount fuer den Live-Snapshot

Gebaeudeaggregation (mehrere Raeume)
------------------------------------
Bei Gebaeuden mit mehreren Raeumen wird vor der Modellierung aggregiert:
1. C_building = Summe aller Raumkapazitaeten
2. pro 5-Minuten-Bin freie Plaetze ueber alle Raeume aufsummieren
3. auf aggregierten Daten Peak + Safe-Arrival berechnen

Damit ist die Empfehlung fuer ein Gebaeude konsistent ueber alle Raeume.

Bayesian Modell pro Zeit-Bin
----------------------------
Mit Prior alpha0=1 und beta0=1:

alpha_t = alpha0 + Summe_i( w_i * free[i,t] )
beta_t  = beta0  + Summe_i( w_i * (C - free[i,t]) )

mit
w_i = exp(-days_ago_i / tau)

Referenzdatum ist der neueste Tag in der Historie, damit "Vorher"-Filter
die letzten Wochentage nicht komplett verwerfen.

Danach wird geglaettet (Moving Average, Fenster=5).

Frueh gewichteter Peak
----------------------
Statt reinem Durchschnittspeak:
1. Pro Tag wird zuerst die maximale Belegung gesucht (peak_occupancy).
2. Der Tagespeak ist der **erste** Zeitpunkt, an dem eine Schwelle des
   Tagespeaks erreicht wird:
   threshold = peak_occupancy * daily_peak_threshold_ratio (Standard 0.85)
   Wir waehlen den fruehesten Bin mit belegten Plaetzen >= threshold.
   (Fallback: falls kein Bin die Schwelle erreicht, den Max-Peak nehmen.)
3. Jeder Tagespeak bekommt Gewicht:
   - recency_weight = exp(-days_ago / tau)
   - early_weight   = 1 + early_peak_bias * (1 - bin/max_bin)
   - total_weight   = recency_weight * early_weight
4. Peak ⌀ wird als gewichtetes Quantil ueber die Tagespeaks berechnet
   (early_peak_quantile, Standard 0.45).

Interpretation:
- fruehere Peaks sind bewusst staerker gewichtet
- spaetere Peaks fliessen weiter ein (Trend bleibt sichtbar)

Peak-Trend
----------
Auf Basis der Tagespeaks wird eine gewichtete lineare Regression gerechnet
(Bin gegen Zeitachse der letzten Tage, mit Recency-Gewichten).

Aus der Steigung in Minuten/Tag:
- > +2.0 min/Tag  => "wird später"
- < -2.0 min/Tag  => "wird früher"
- sonst           => "bleibt gleich"

Safe-Arrival Entscheidung
-------------------------
Sicherheitslogik:
- buffer_seats = max(ceil(buffer_ratio * C), 1)
- k_eff = min(C, k + buffer_seats)

Pro Bin:
- P_t = P(X >= k_eff)
- X folgt Beta-Binomial-Predictive (numerisch stabil im Log-Space)

Wichtig:
- Suchbereich ist nur [Oeffnung, Peak ⌀ - lead_time_before_peak]
- Dadurch wird keine Empfehlung exakt am Peak oder danach ausgegeben

latest_safe_time = spaetester Bin im Suchbereich mit:
- Ort ist offen
- ausreichende Datenstuetzung
- P_t >= p_goal

Ausgabe
-------
- Empfohlene Ankunft bis HH:mm (oder "keine sichere Zeit")
- Peak ⌀
- Sicherheitsniveau (hoch / mittel / niedrig)
- Peak-Trend
- interne Kennzahlen: probability, expected_free_seats, confidence_flag,
  peak_occupancy_rate, peak_trend_minutes_per_day
